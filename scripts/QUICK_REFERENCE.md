# Migration Script Quick Reference

## One-Line Commands

```bash
# Preview changes (SAFE - no modifications)
./scripts/migrate_effect_to_context.sh --dry-run

# Run migration (creates backup automatically)
./scripts/migrate_effect_to_context.sh

# Rollback to previous state
./scripts/migrate_effect_to_context.sh --rollback

# Test the script (recommended first step)
./scripts/test_migration.sh
```

## What Gets Changed

### ✅ These Will Be Replaced

| Old | New |
|-----|-----|
| `EffectSet` | `ContextSet` |
| `EffectInferenceContext` | `ContextInferenceEngine` |
| `Effect` (type) | `Context` |
| `has_effect()` | `has_context()` |
| `add_effect()` | `add_context()` |
| `add_effects()` | `add_contexts()` |
| `get_effects()` | `get_contexts()` |
| `take_effects()` | `take_contexts()` |
| `effect_set` | `context_set` |
| `current_effects` | `current_contexts` |
| `effects.rs` | `contexts.rs` |

### ✋ These Will Be Preserved

| Pattern | Reason |
|---------|--------|
| `side effect` | General CS term |
| `side-effect` | General CS term |
| `no side effect` | Common phrase |
| `memory effect` | FFI concept |
| `FFI effect` | FFI-specific |

## Quick Workflow

```bash
# 1. Test first (optional but recommended)
./scripts/test_migration.sh

# 2. Check current state
git status  # Ensure clean working tree
cargo test --all  # Ensure baseline passes

# 3. Preview changes
./scripts/migrate_effect_to_context.sh --dry-run

# 4. Run migration
./scripts/migrate_effect_to_context.sh

# 5. Review results
cat migration_report_*.md
git diff --stat

# 6. Validate manually
grep -r "side context" crates/  # Should be empty
cargo check
cargo test --all

# 7. Commit or rollback
git add -A && git commit -m "refactor: effect → context"
# OR
./scripts/migrate_effect_to_context.sh --rollback
```

## Common Patterns

### Before & After Examples

**Type Definition:**
```rust
// Before
pub enum Effect { Pure, IO, Async }
pub struct EffectSet { effects: Set<Effect> }

// After
pub enum Context { Pure, IO, Async }
pub struct ContextSet { contexts: Set<Context> }
```

**Method Calls:**
```rust
// Before
ctx.add_effect(Effect::IO);
let effects = ctx.get_effects();

// After
ctx.add_context(Context::IO);
let contexts = ctx.get_contexts();
```

**Imports:**
```rust
// Before
use verum_types::effects::{Effect, EffectSet};

// After
use verum_types::contexts::{Context, ContextSet};
```

**Preserved:**
```rust
// These stay unchanged
/// Pure functions have no side effects
fn is_pure() { }

// Memory effects at FFI boundary
pub enum MemoryEffect { Read, Write }
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Permission denied | `chmod +x scripts/migrate_effect_to_context.sh` |
| Compilation fails | `./scripts/migrate_effect_to_context.sh --rollback` |
| Want to re-run | Use `--no-backup` flag |
| Unexpected changes | Check exclusion patterns, rollback if needed |

- ✅ No `EffectInferenceContext` in code

- ✅ No "side context" false replacements
