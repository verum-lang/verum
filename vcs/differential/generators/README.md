# Differential Test Generators

This directory contains tools for generating test programs for differential testing between Tier 0 (interpreter) and Tier 3 (AOT).

## Files

### random_program_generator.rs

A Rust module that generates syntactically valid random Verum programs. Features:

- **Grammar-aware generation**: Produces valid Verum code based on `grammar/verum.ebnf`
- **Configurable complexity**: Control depth, size, and feature usage
- **Deterministic via seed**: Same seed produces same program
- **Characteristic generators**: Specialized generators for specific patterns
  - Recursive programs
  - Nested loops
  - Complex expressions
  - Pattern matching

## Usage

### Basic Random Program

```rust
use crate::generators::random_program_generator::{RandomProgramGenerator, GeneratorConfig};

// Generate with default config
let mut gen = RandomProgramGenerator::with_seed(42);
let program = gen.generate();

// Generate with custom config
let config = GeneratorConfig {
    seed: 12345,
    max_depth: 8,
    max_statements: 30,
    enable_floats: true,
    enable_closures: true,
    ..Default::default()
};
let mut gen = RandomProgramGenerator::new(config);
let program = gen.generate();
```

### Characteristic Generators

```rust
use crate::generators::random_program_generator::CharacteristicGenerator;

let mut gen = CharacteristicGenerator::new(42);

// Generate a program with deep recursion
let recursive_program = gen.generate_recursive();

// Generate a program with nested loops
let loop_program = gen.generate_nested_loops();

// Generate complex expression tests
let expr_program = gen.generate_complex_expressions();

// Generate pattern matching tests
let pattern_program = gen.generate_pattern_matching();
```

### Batch Generation

```rust
use crate::generators::random_program_generator::{generate_batch, generate_characteristic_batch};
use std::path::Path;

// Generate 100 random programs
let paths = generate_batch(100, 42, Path::new("output/random"))?;

// Generate characteristic test suite
let paths = generate_characteristic_batch(42, Path::new("output/characteristic"))?;
```

## Configuration Options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `seed` | u64 | 42 | Random seed for reproducibility |
| `max_depth` | usize | 5 | Maximum expression nesting depth |
| `max_statements` | usize | 20 | Maximum statements per function |
| `max_functions` | usize | 5 | Maximum helper functions |
| `max_loop_iterations` | usize | 100 | Maximum loop iterations |
| `max_recursion_depth` | usize | 10 | Maximum recursion depth |
| `enable_floats` | bool | true | Enable floating-point operations |
| `enable_async` | bool | false | Enable async operations |
| `enable_generics` | bool | false | Enable generic types |
| `enable_closures` | bool | true | Enable closures |
| `enable_collections` | bool | true | Enable List, Map, Set |
| `enable_heap` | bool | true | Enable Heap<T> allocation |

## Generated Program Structure

Each generated program includes:

1. **Test annotations**: `@test`, `@tier`, `@level`, `@tags`
2. **Helper functions**: 0-5 randomly generated functions
3. **Main function**: With statements and output
4. **Deterministic output**: Uses `println` for comparison

## Example Output

```verum
// @test: differential
// @tier: 0, 3
// @level: L1
// @tags: generated, seed_42
// Generated by RandomProgramGenerator, seed: 42

fn fn_1(p0: Int, p1: Bool) -> Int {
    let x_1 = (p0 + 10);
    if p1 {
        let x_2 = (x_1 * 2);
    }
    (p0 - 5)
}

fn main() {
    let x_3 = 42;
    let mut x_4 = true;
    for i_1 in 0..5 {
        let x_5 = (x_3 + i_1);
    }
    println(f"x_3 = {x_3}");
    println(f"x_4 = {x_4}");
}
```

## Integration with Differential Testing

The generated programs are designed for differential testing:

1. **Reproducible**: Same seed = same program
2. **Deterministic output**: No random values in output
3. **Bounded complexity**: Won't hang or stack overflow
4. **Type-correct**: Should compile and run

When a generated program produces different results between tiers:

1. Record the seed and config
2. Use shrinking to minimize the test case
3. Add to regression suite
4. Fix the discrepancy
